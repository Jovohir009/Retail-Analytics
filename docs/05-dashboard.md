# Dashboard

## Overview

The Dashboard is the web application used by customers to monitor visitor analytics generated by the AI Agent.

It provides real-time and historical insights about customer traffic without requiring access to CCTV cameras or video streams.

The dashboard is accessible through any modern web browser after user authentication.

The dashboard never communicates directly with cameras or AI Agents.

All information is provided by the backend.

---

# Responsibilities

The Dashboard is responsible for:

- User authentication
- Displaying visitor analytics
- Visualizing reports
- Displaying camera status
- Managing store settings
- Managing camera settings
- Displaying AI Agent status

The dashboard does not perform AI processing or business calculations.

---

# Dashboard Structure

The MVP dashboard consists of the following pages.

- Login
- Dashboard (Home)
- Analytics
- Cameras
- Settings
- Profile

---

# Login

The login page allows users to securely access their account.

Responsibilities:

- User authentication
- Remember user session
- Redirect authenticated users

---

# Dashboard (Home)

The dashboard home page provides a quick overview of the current business status.

Displayed information:

- Today's visitors
- Live visitor count
- Visitor trend
- Peak hour
- Camera status
- AI Agent status

This page is designed for quick decision making.

---

# Analytics

The Analytics page displays historical visitor information.

Available reports:

- Hourly visitors
- Daily visitors
- Weekly visitors
- Monthly visitors

Charts should be interactive and easy to understand.

Users should be able to compare different time periods.

---

# Cameras

The Cameras page allows users to manage connected cameras.

Displayed information:

- Camera name
- Camera status
- Camera type
- Last synchronization
- Connection status

For MVP:

One camera is supported.

Future versions may support multiple cameras.

---

# Settings

The Settings page allows users to manage application preferences.

Examples:

- Store information
- Time zone
- Notification preferences
- Language

Camera credentials should never be displayed.

---

# Profile

The Profile page allows users to manage their personal account.

Available actions:

- Update profile
- Change password
- Logout

Future versions may support profile pictures and additional account settings.

---

# Dashboard Widgets

The homepage should display important information using widgets.

Examples:

## Today's Visitors

Displays the total number of visitors that entered today.

---

## Live Visitor Count

Displays visitor entries in near real time.

---

## Peak Hour

Displays the busiest hour of the day.

---

## Visitor Trend

Shows whether visitor traffic is increasing or decreasing compared to previous periods.

---

## Camera Status

Displays whether the connected camera is:

- Online
- Offline

---

## AI Agent Status

Displays whether the AI Agent is:

- Running
- Offline
- Synchronizing

---

# Data Visualization

Analytics should be displayed using charts instead of raw numbers whenever possible.

Preferred visualizations:

- Line charts
- Bar charts
- Summary cards

Charts should remain simple and readable.

The MVP should avoid overly complex visualizations.

---

# Real-Time Updates

The dashboard should automatically update when new visitor events arrive.

Users should not need to manually refresh the page.

Only affected widgets should update to minimize unnecessary rendering.

---

# User Experience

The dashboard should prioritize simplicity.

Design principles:

- Clean interface
- Fast loading
- Responsive layout
- Consistent navigation
- Minimal clicks
- Clear typography

The dashboard should be usable without technical knowledge.

---

# Responsive Design

The dashboard should support:

- Desktop
- Laptop
- Tablet
- Mobile

The MVP is optimized primarily for desktop usage while remaining functional on smaller devices.

---

# Performance Goals

The dashboard should provide:

- Fast page loading
- Smooth navigation
- Near real-time updates
- Responsive interactions
- Efficient rendering

Large datasets should not negatively affect user experience.

---

# Security

The dashboard should:

- Require authentication
- Protect user sessions
- Restrict unauthorized access
- Never expose sensitive system information
- Never expose camera credentials

All communication with the backend should use secure HTTPS connections.

---

# Future Features

The dashboard architecture should support future additions without redesign.

Potential future features include:

- Multi-store management
- Multi-camera management
- Occupancy dashboard
- Heatmaps
- AI-generated insights
- Report export (PDF, Excel)
- Email reports
- Notifications
- Mobile application support

---

# Design Principles

The dashboard should follow these principles:

- Simple
- Modern
- Responsive
- Accessible
- Easy to navigate
- Fast
- Scalable
- Production-ready

The interface should help users understand their business at a glance without requiring technical expertise.
